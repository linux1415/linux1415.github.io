How does this system work?

Any key that is created/used by this system is, essentially, a very massive polyalphabetic substitution cipher. A substitution cipher is a cipher where there is a ciphertext alphabet and the traditional alphabet; when encryption is done, each letter in the plaintext is replaced with the corresponding character from the ciphertext alphabet. In other words, all a's would become a specific character, all b's would become a specific character.

This is an example of a simple substitution cipher where every letter in the alphabet is represented with only one character string. This is not secure and will be broken easily. A polyalphabetic substitution cipher is where each letter of the alphabet (and numbers, etc.) is represented with numerous possibilites. For instance, all a's could be represented by character strings such as "7fyrh", "yeT5W", "W4nY3", etc, etc. So just one single letter could be represented by 10's, 100's, or even 1000's of possibilites. This is the way the keys in this system work; each plaintext character - whether it be "a", "#", "8", "W", or any other of the possibilites - will be represented with a different ciphertext character string when encryption is done.

The digital keys used for encryption in this system are similar to the above, except instead of only 10 ciphertexts per plaintext character, there are hundreds or thousands. Also, when an individual ciphertext character string is used to represent a plaintext character, the ciphertext string is not used again in the message; this feature will make any message unbreakable if key is used only once. Each subsequent time a key is used, it will get slightly (very slightly) less secure; to mitigate this, level two encryption can be used. Level two encryption encrypts the ciphertext further based on a number code and passcode. The encryption used with level two is similar to that of a one time pad. Level two encryption allows one to use the same key many, many times as long as passcode and number code are changed frequently.
